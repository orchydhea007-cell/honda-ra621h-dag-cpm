Algorithm Pipeline

O(V + E)
01

Build DAG

103 components modelled as nodes. Each dependency entry becomes a directed edge. Adjacency list + reverse adjacency built simultaneously. 16 root nodes (no predecessors), C103 as the single sink.

02

Kahn's Topological Sort

BFS-based sort: compute in-degree for each node, enqueue zeros, process in waves. Each wave represents a parallelisable batch. Detects cycles via residual in-degree > 0. Complexity: O(V + E).

03

CPM — Forward Pass

Traverse topological order, compute ES = max(EF of predecessors) and EF = ES + duration for each node. Project duration = max EF across all leaf nodes.

04

CPM — Backward Pass

Traverse reverse order, compute LF = min(LS of successors) and LS = LF − duration. Total Float TF = LS − ES. Nodes with TF = 0 form the critical path.

How Kahn's Algorithm Works

1

Compute In-Degree

For each node, count incoming edges. Nodes with in-degree = 0 have no prerequisites — they go into the initial queue.

2

Process Queue

Dequeue a node, add it to the sorted order. For each of its successors, decrement their in-degree. When a successor's in-degree hits 0, enqueue it.

3

Cycle Detection

If after processing, the output contains fewer nodes than the graph — a cycle exists. For assembly scheduling, this means a circular dependency (impossible to assemble).

4

Parallel Batching (Waves)

Nodes processed in the same BFS level form a "wave" — they can be assembled simultaneously by different teams. This directly maps to resource scheduling.

CPM Interpretation Guide

ES

Earliest Start

The earliest time this component can begin assembly — determined by when all its prerequisites finish. ES = max(EF of all predecessors).

EF

Earliest Finish

Earliest completion time. EF = ES + Duration. Drives the forward pass — each node's EF gates its successors' ES.

LS

Latest Start

The latest this component can start without delaying the project. LS = LF − Duration. Computed in backward pass (right-to-left).

TF

Total Float

Scheduling flexibility. TF = LS − ES. If TF = 0, the component is on the critical path — any delay cascades to the final deadline.

Algorithm Explainer

DAG + Kahn's Topological Sort + Critical Path Method

Why DAG?

A Directed Acyclic Graph (DAG) perfectly models assembly scheduling because:

→

Directed Edges = Dependencies

An edge from A → B means "A must be assembled before B". The direction encodes the mandatory ordering constraint.

∅

Acyclic = No Circular Deps

A cycle would mean "A needs B, B needs A" — physically impossible. DAGs guarantee a valid assembly order always exists.

⊕

Topological Order = Build Sequence

Any topological ordering of the DAG is a valid assembly sequence. Multiple valid orderings → scheduling flexibility.
